What is the EKS Platform Hub?

A portal for project leads to manage access to the ACP Elastic Kubernetes Service (EKS) sandbox kubernetes cluster. If you are looking for the live KOPs based environments please visit the live instance of platform hub located here.

How do I create a user account on the Hub?

This is the same process as the live environment, get the user to Login (Sidebar > Login) using their single sign-on account (SSO) through Keycloak and a new account will be created for them on the Hub

How can I create a namespace on the sandbox cluster?

Please raise a support ticket requesting the namespace be created on the new EKS cluster.

Where is (x) feature of platform hub?

Many features of platform hub have been disabled in this version of platform hub. This is a sandbox environment dedicated to the single EKS notprod sandbox.

How do I connect to the cluster?

Get one of your project admins to allocate you to a RBAC group for your project. These will either be read-only, pod-management or admin scoped to the project's namespaces. To connect to the cluster you can utilise the helper script located in this github repo. If for some reason you cannot use the script please directly login to the vault UI under the cluster-auth provider and generate credentials for the role you have been allocated in this platform hub instance.

Where are the old tokens?

The static tokens used by the KOPs clusters cannot be used in EKS. As a result we needed to design a new solution for authenticating with the cluster. More information on the authentication architecture can be found in the EKS Docs.

Does this mean just in time escalation has been removed?

The just in time escalation solution has been left in place. As this is a Sandbox environment this can be removed at a tenants request. The solution has some quirks and interactions with Vault that are unintended, because this solution will not be productionised we have avoided implementing a fix.

Why are tenant RBAC groups no longer scoped to a namespace?

As this architecture is no longer going to progress to production, we decided to keep the RBAC roles tenant scoped. It might have caused technical issues to give more granular permissions.
